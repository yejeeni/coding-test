## Day 3 — 직접 구현 1: 배열 기반 스택과 큐
### 🟢 문제 1 — 내 스택으로 괄호 검사 (워밍업)

**[백준 스타일]**

**제한**
- 시간: 1초 / 메모리: 256MB
- 1 ≤ 문자열 길이 ≤ 100,000
- 문자열은 `(`, `)` 로만 구성

**문제**
괄호 문자열이 주어진다. 괄호가 올바르게 짝지어져 있으면 `YES`, 아니면 `NO`를 출력하라.

단, **`ArrayDeque`·`Stack` 등 Java 컬렉션을 사용하지 말고**, `int[]` 배열과 top 인덱스로 스택을 직접 구현할 것.

**입력 형식**
```
한 줄: 괄호 문자열
```

**출력 형식**
```
YES 또는 NO
```

**예제 입력 1**
```
((()))
```
**예제 출력 1**
```
YES
```

**예제 입력 2**
```
(())(
```
**예제 출력 2**
```
NO
```

**예제 입력 3**
```
)(
```
**예제 출력 3**
```
NO
```

> 💡 구현 힌트: 스택에 실제로 문자를 저장할 필요 없어요. `(` 개수만 추적하면 돼요. top 인덱스가 음수가 되는 순간이 어떤 의미인지 생각해보세요.
