### 🟡 문제 2 — Lower / Upper Bound (메인)

**[백준 스타일]**

**제한**
- 시간: 1초 / 메모리: 256MB
- 1 ≤ N ≤ 100,000
- 배열은 오름차순 정렬되어 주어짐
- 1 ≤ 각 수, 쿼리 수 ≤ 1,000,000

**문제**
정렬된 N개의 수와 쿼리 수 K가 주어진다.
- lower bound: K가 처음 등장하는 인덱스 (없으면 K보다 큰 수가 처음 등장하는 인덱스)
- upper bound: K보다 큰 수가 처음 등장하는 인덱스

두 값을 공백으로 구분해 출력하라.

**입력 형식**
```
첫째 줄: N
둘째 줄: N개의 정수 (오름차순)
셋째 줄: K
```

**출력 형식**
```
lower_bound upper_bound
```

**예제 입력 1**
```
7
1 2 2 2 3 4 5
2
```

**예제 출력 1**
```
1 4
```

**예제 입력 2**
```
5
1 3 5 7 9
4
```

**예제 출력 2**
```
2 2
```

💡 체크포인트: lower bound → arr[mid] < K이면 lo=mid+1 / upper bound → arr[mid] <= K이면 lo=mid+1. 딱 한 글자 차이예요.
